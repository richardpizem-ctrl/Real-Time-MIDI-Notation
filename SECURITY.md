# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported |
|---------|-----------|
| Latest  | Yes       |
| Older   | No        |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

To report a vulnerability, you may:

- open an **Issue** on GitHub, or  
- contact the maintainer directly at **richardpizem@gmail.com**

Please include:

- a clear description of the issue  
- steps to reproduce  
- potential impact  
- any logs, screenshots, or proof‑of‑concept code (if available)

All reports will be reviewed as soon as possible.  
Responsible disclosure helps keep the project safe for everyone.

---

## Responsible Disclosure Expectations

When reporting a vulnerability, please:

- avoid publicly posting details before it is fixed  
- allow reasonable time for investigation  
- avoid exploiting the vulnerability beyond what is necessary to demonstrate it  

We appreciate your help in keeping **Real‑Time MIDI Notation** secure.

## Thank You

Security researchers, testers, and contributors play a crucial role in improving this project.  
Your effort and responsible reporting help protect users worldwide.
